    • Introduction to HTML

    Every website is based on Hypertext Markup Language (HTML).
    HTML defines the content and structure of web content.
    A web browser is what translates HTML code into web pages.

    It will allow you to:

    • Create websites
    • Work more effectively with technical teams
    • Improve digital marketing skills

    You can then style the page with CSS.
    JavaScript is used to make pages interactive.

    1. Basic Tags & Elements

    HTML code is based on tags using angel brackets: "< >".
    These angel brackets surround the name of the element you want to add to the page:

    <button>

    Many HTML elements require both opening and closing tags to indicate start and end.
    These are called container tags, otherwise they are just empty tags.

    Examples:

    Button: <button> Like </button>:

    Image: <img src = "forest2.png" >:

    Text paragraph: <p> Hello World! </p>: Hello World!

    Table: <table> </table>

    2. Headings

    Headings in HTML come in different levels. The most important one being h1.
    They are another element requiring opening and closing container tags.

    <h1> Heading 1 <h1>:

    Heading 1

    You can use up to 6 levels of headings in HTML. The tags for these heading elements are <h1>, <h2>, <h3>, <h4>, <h5> and <h6>.

    <h1> is usually styled with the largest font, and <h6> with the smallest.

    <h6> Heading 6 <h6>:

    Heading 6

    3. Comments & Readability

    Comments help humans to read your code. You can add them with the comment tag:
    <!-- … -->

    You can also use comments to temporarily disable parts of your code:
    <!--<button> OK </button>-->

    Since HTML is a case-insensitive language, the web browser understands tags in both upper and lowercase form.

    The web browser will also ignore white spaces and line breaks in your code.

    It's still good practice to keep an organized structure for future work.

    e.g.

    <p>Welcome!
    Enter your name</P>

    will still output: Welcome! Enter your name

    In order to create a line break, the <br> tag will force a line break:

    e.g.

    <p>First line of text.<br> Second line of text:</p>

    First line of text.
    Second line of text

    Elements like headings and paragraphs automatically start on a new line.

    4. Standards and best practices

    are needed to make:

    1. web pages compatible with different browser and devices
    2. teamwork and collaboration more efficent

    The <body> container tag is used to group everything that gets displayed on a page when loaded in a browser.

    
            <body>
            <h1> Test1 </h1>
            <p> Test2 </p>
            </body>

    prints:

    Test1

    Test2

    Body tags are needed to be compatible with all web browser.
    A web page can only contain one body element.
    All the content elements you need to display need to be inside the <body> container.
    When HTML tags go inside others, this is called nesting.

    Websites do more than just display content.
    The <head> container tag is used to include technical information about the page:

    
            <head>
            <title>HTML</title>
            <meta name="keywords" content="html, code">

    Title, description and keywords are nested in the head tag.

    The head needs to be placed before the body.
    The information in the head is not displayed on the web page, only the title is shown in search engines and the browser tab.

    Today most wrap their HTML code in <html> container tags. Head and body are nestes inside the <html> tags.

    5. Text formatting

    Helps visitors find the information they are looking for.
    It's used to highlight the important content.
    Formatting tags are container tags.

    The bold tag <b> is used to display text in bold:
    <b>Bold text</b>: Bold Text

    The italic tag <i> is used to display text in italics:
    <i>Italic text</i>: Italic Text

    The underline tag <u> is used to underline text:
    <u>Underlined text</u>: Underlined Text

    Formatting tags are applied to text and are nested inside elements.

    e.g.

    <p> Movie: <i> E.T. </i> </p>:

    Movie: E.T.

    Accessibility

    There are some HTML formatting tags you can use to make your web pages more accessible.

    The strong tag <strong> is used to highlight important text:
    <strong>Important text</strong>: Important text

    Strong text is displayed in bold, just like the <b> tag.
    The difference is that the <strong> tag also has meaning and is used by screen readers.

    The emphasis tag <em> is used to define emphasized text:
    <em> Emphasized text </em>: Emphasized text

    Emphasized text is displayed in italic, just like the <i> tag.
    The difference is that the screen reader will verbally stress the words.

    The <strong> and <em> tags are considered semantic formatting tags because they add meaning to the content.
    They can serve as an indication of emphasis to a screen reader.

    6. Links

    HyperText is what lets people jump from one web page to another, through links.
    The Web woudln't be possible without it.
    HyperText is text that contains a link to another page.
    Web pages are called HyperText documents because they are connected by Hypertext links (or hyperlinks).
    Hyperlinks allow users to move from one web page to another and share information across different machines and systems.

    The anchor container tag <a> is used to create a hyperlink on a web page.

    To create a link, you need "href" (hypertext reference) to add the destination URL:
    <a href = "https://alintras.github.io/">Link</a>: Link

    The URL needs to be enclosed in macthing quotes (') or (") to work without errors.
    If you forget to add href or the quotes the <a> tag won't create a hyperlink.

    You can also use hyperlinks to connect pages of the same website.
    The part of the URL that stays the same for all pages of a website is called the domain name.

    7. Lists

    Lists are useful elements, even used to build navigation menus.

    A list consists of list items <li>

    e.g.

    
                <li> Item 1 </li>
                <li> Item 2 </li>
                <li> Item 3 </li>

    Ordered Lists <ol> show with numbers instead of bullet points
    e.g.

    
                <ol>
                <li>Coffee</li>

                <li>Milk</li>
                </ol>

    1. Coffe

            <ul>
            <li>Coffee</li>
            <li>Tea</li>
            <li>Milk</li>
            </ul>

    results in:

    • Coffe
    • Tea
    • Milk

    Of course you can also nest a list inside another list.

    8. Attributes

    You can have more control over HTML elements with attributes.
    They are modifiers that provide additional information about elements.

    Attributes can be optional or required. The Image tag requires an attribute to work correctly.

    <img src = "http://www.img.png">

    For container tags, attributes always go in the opening tag
    the anchor tag is another example of an HTML element that requires an attribute to work correctly.

    <a href = "https://z.com">Z</a>

    href and src are examples of attributes.

    It's best practice to describe what images are showing
    The alt attribute (alternative text) is used to add image descriptions.

    <img src="forest2.png" alt="a green forest">

    The alt attribute can be:

    • read aloud by screen readers
    • shown when the image doesn't load
    • read by search engines

    Attributes usually come in name/value pairs:

    name="value"

    e.g.
    Attribute name: src
    Attribute value:"http://www.image.png"

    You can control the size of images in your web page.
    width is an optional attribute:
    <img src="forest.png" width="300">

    height is another optional attribute for the image element.
    <img src="forest.png" height="150">


    or together:
    <img src="forest.png" width="200" height="130>

    When only one of the 2 attributes is given, the web browser will calculate the other based on the aspect ratio, so the image is not stretched or squeezed.
    Changing the aspect ratio of an image causes distortion.

    9. Menus & Navigation

    The way visitor navigate a website depends on its design.
    Websites come in 2 shapes:

    • Multi-page websites
    • Single-page websites
    Navigation has a big impact on user experience, Search Engine rankings and therefore sales.

    Multi-page websites are made of more than one page.
    Navigating between pages requires the web browser to load different HTML files.

    The <nav> container tag defines a set of links that allows the user to navigate between pages of a website:

                    <nav>
                <a href="index.html">Home</a>
                <a href="Maths.html">Maths</a>
                <a href="Coding.html">Coding</a>
                </nav>
    results in:
    Home Maths Coding

    A single-page website has all its content on the home page. Any navigation links take the visitor down to different sections (instead of loading new pages).
    (It's best practice to name your homepage index.html so that the web browser can find and load it)

    To jump to a specific part of a single-page website, first you need to mark the section with the id (ID) attribute.

                        <h2 id ="section-1">Section 1</h2>
                    <section id="menus-navigation">

    The id attribute is used to identify the element you want to target with the navigation link.
    Each id value must be:

    • unique
    • within quotes

    Once the element you want to jump to has been marked with an id, you can target it with an anchor tag <a>

    <a href = "#menus-navigation">Go to Navigation</a>

    results in:

    Go to Navigation

    The hash character (#) is needed to tell the web browser that we are targeting a section of the same page.

    13. Videos

    You can use the <video> container tag to embed video files into a web page. 

                    <video>
                    <!--video file-->
                </video>

    Just like with images, the source (or location) URL of the video needs to be provided.

    Common video formats are: MP4, OGG and WebM

                    <video>
                    <source src="file.mp4" type ="video/mp4">
                </video>

    The <source> empty tag provides the location and format of the video file.
    The src attribute specifies the URL of the video file.
    The type attribute specifies the format of the video file.

    Different media format are needed for compatibility with different browser and devices.
    The <source> tag allows you to add multiple files. 

                    <video>
                    <source src="file.mp4" type ="video/mp4">
                    <source src="file.webm" type ="video/webm">
                </video>

    When different <source> options are included, the browser will choose the first one it supports.
    You can add text between the <video> tags. The text will only be displayed in browsers that don't support the video element. 

                    <video>
                    <source src="file.mp4" type ="video/mp4">
                    <source src="file.webm" type ="video/webm">
                    Video not supported
                </video>

    You can display play/pause, volume and other default video controls with the controls attribute.

                    <video controls>
                    <source src="file.mp4" type ="video/mp4">
                V   ideo not supported
                </video>

    15. Layout

    The body of a web page can be divided into 3 parts.

    • Header: <header>
    • Main content: <main>
    • Footer: <footer>

    The <header> container tag usually contains introductory information.
    The header often contains navigation links, a menu and/or a search bar.
    Brand elements like logis are usually found in the header too.
    You can add several header elements to a web page. 

                <header>
            <!--Introductory information-->
            </header>

    The <main> container tag is used to include the main content of a web page.
    There must not be more than one <main> element in a document. 

                <main>
            <!--main content-->
            </main>

    The <footer> container tag often contains contact and legal information and links.
    You can have several <footer> elements in one document. 

                <footer>
            <!--footer information-->
            </footer>

    Futhermore, <header>, <main> and <footer> elements are nested inside the <body> container tag 

                <body>
                <header>
                    <!--intro information--> 
                </header>
                <main>
                    <!--main content-->
                </main>
                <footer>
                    <!--footer information-->
                </footer>
            </body>

    Don't confuse <header> with <head>.
    <head> is used to include technical information about the page like title, description, author and keywords and not shown to the user.

    Building a website involves two areas:

    Front-end: What users see and interact with. Back-end: Information requests, servers and databases.

                <p style="font-size:large">Large paragraph</p>

    8. Attributes

    You can have more control over HTML elements with attributes.
    They are modifiers that provide additional information about elements.

    Attributes can be optional or required. The Image tag requires an attribute to work correctly.

    <img src = "http://www.img.png">

    For container tags, attributes always go in the opening tag
    the anchor tag is another example of an HTML element that requires an attribute to work correctly.

    <a href = "https://z.com">Z</a>

    href and src are examples of attributes.

    It's best practice to describe what images are showing
    The alt attribute (alternative text) is used to add image descriptions.

    <img src="forest2.png" alt="a green forest">

    The alt attribute can be:

    • read aloud by screen readers
    • shown when the image doesn't load
    • read by search engines

    Attributes usually come in name/value pairs:

    name="value"

    e.g.
    Attribute name: src
    Attribute value:"http://www.image.png"

    You can control the size of images in your web page.
    width is an optional attribute:
    <img src="forest.png" width="300">

    height is another optional attribute for the image element.
    <img src="forest.png" height="150">


    or together:
    <img src="forest.png" width="200" height="130>

    When only one of the 2 attributes is given, the web browser will calculate the other based on the aspect ratio, so the image is not stretched or squeezed.
    Changing the aspect ratio of an image causes distortion.

    18. Inline and Block Elements

    Every HTML element is either inline or block.
    This is one of the most important things for web designers and developers.

    The border property is great for helping us see the difference between these 2 types of HTML elements. 

                <button style="border:solid">Click me</button>

    The borders of the different elements will help you spot the difference. 

                <h2>Block-Level Elements</h2>
            <!--Start on a new line-->
                <h3 style="border:solid">Heading</h3>
                <p style="border:solid">Paragraph</p>
                <ul style="border:solid">
                    <li>List item A</li>
                    <li>List item B</li>
                </ul>
            <h2>Inline Elements</h2>
            <!--Don't start on a new line-->
                <p><b style="border:solid">Bold</b> text</p>
                <button style="border:solid">Click me</button>
                <label style="border:solid">Label</label>

    Block-Level Elements

    Heading

    Paragraph

    • List item A
    • List item B

    Inline Elements

    Bold text

    Block-level elements always start on a new line. They take up the full width available.
    Inline elements don't start on a new line. They only take up as much width as necessary.

    Format tags are inline elements:

    • Bold:<b>
    • Italics:<i>
    • Underlined:<u>

    Inline elements can be nested inside block-level elements.

    Line break <br> tags are used to used to force inline elements to start on a new line.

    Line break tags are usually included in forms, because form elements like <input> and <label> are inline elements.

    Divide & Conquer

    Grouping different HTML elements can make your pages faster to load and easier to customize and maintain.

    The <div> element is a container for HTML elements that keeps your pages organized.

    The <div> element always takes up the full width available.
    This means the <div> element is a block-level element.
    This means it starts on a new line.

    Elements grouped in a <div> can be styled all at once.

                <div style="text-align:center">
                <h2>About Me</h2>
                <p>Hi, I'm Ali</p>
                <img src="forest.png" width=50>
            </div>

    About me

    Hi, I'm Ali

    <div> itself doesn't add any visual effects unless you add a style to it.
    It's often used to group and style HTML elements.

    The style in a <div> container will apply to all its nested elements unless you give them their own.
