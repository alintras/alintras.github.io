Git

My personal documentation for git fundamentals

Git is a tool used to keep track of changes to files, especially the code of the projects.
It is termed a distributed version control system because of its behaviour to allow multiple people to work on the same project, even if they are not connected to a common server.

CommandDescription
git clone URLcopies a repository from GitHub to your computer
git pull origin mainupdates your local repository with latest changes from GitHub
git statusshows which files were changed
git add .adds all changes to the next commit
git commit -m "message"saves your changes locally with a message
git push origin mainuploads local commits to GitHub
git reset --harddiscards all local changes since the last commit
git clean -fddeletes untracked files and folders
git fetch origindownloads info about changes on GitHub without merging them
git reset --hard origin/mainmakes your local copy exactly match the version on GitHub
git branchlists all branches
git checkout -b new-branchcreates and switches to a new branch
git merge branchnamemerges another branch into the current one
