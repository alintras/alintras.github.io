Git

My personal documentation for git fundamentals

Git is a tool used to keep track of changes to files, especially the code of the projects.
It is termed a distributed version control system because of its behaviour to allow multiple people to work on the same project, even if they are not connected to a common server.

Command Description
git clone URL copies a repository from GitHub to your computer
git pull origin main updates your local repository with latest changes from GitHub
git status shows which files were changed
git add . adds all changes to the next commit
git commit -m "message" saves your changes locally with a message
git push origin main uploads local commits to GitHub
git reset --hard discards all local changes since the last commit
git clean -fd deletes untracked files and folders
git fetch origin downloads info about changes on GitHub without merging them
git reset --hard origin/main makes your local copy exactly match the version on GitHub
git branch lists all branches
git checkout -b new-branch creates and switches to a new branch
git merge branchname merges another branch into the current one
